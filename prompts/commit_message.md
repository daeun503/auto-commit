중요 규칙:
- 중간 사고 과정(thinking, reasoning, analysis)을 출력하지 마라
- 설명하지 말고 결과만 출력하라
- 반드시 JSON 배열만 출력하라 (추가 텍스트 금지)
- 커밋 메시지는 한글로 작성하되 영어 단어(특히 기술 용어)는 그대로 사용하라.
- 하단 출력 형식을 반드시 지켜라

목표:
- 커밋 메시지 후보 5개를 생성하라.
- 아래 diff는 여러 종류의 변경(기능 추가, 버그 수정, 리팩터링 등)이 섞여 있을 수 있다.
- 그러므로 커밋 메시지 후보 5개는 "여러 type이 섞여도 된다".
- 단, 각 후보 메시지는 반드시 하나의 type만 사용해야 하며, 한 메시지 안에서 type을 섞지 마라.

후보 생성 규칙:
1) diff 전체를 보고 변경 성격을 2~4개 범주로 요약하라 (feat/fix/refactor/test/chore 중)
2) 각 범주에 대해 1개 이상 후보를 만들되, diff에서 비중이 큰 범주에 더 많은 후보를 배정하라
   - 예: feature 변화가 크고 fix는 작으면 (feat 3~4개, fix 1~2개)
3) 후보들은 서로 표현이 겹치지 않게 다양하게 작성하라
4) 한 줄 커밋 메시지 형식:
   "<gitmoji> <type>: <짧은 설명>"

gitmoji 매핑:
- feat      → ✨
- fix       → 🐛
- refactor  → ♻️
- test      → 🧪
- chore     → 🧹

출력 형식 (반드시 이 형태):
[
  "✨ feat: ...",
  "🐛 fix: ...",
  "♻️ refactor: ...",
  ...
]
